time.mattrude.com — NTP Pool Cluster

Welcome to my public time server. This site, time.mattrude.com, is a cluster of three ntp time servers hosted by Rackspace in Dallas TX.

These servers are part of the NTP Pool and are avalable to the public for synchronizing computer time over the internet. With these Network Time Protocol (NTP) servers, you should be able to synchronize most system clocks when properly configured.

Please do not directly configure a system to use only these servers.

NTP Pool

The NTP Pool project is a big virtual cluster of time servers providing reliable, easy to use NTP service for millions of clients.

The pool is being used by millions or tens of millions of systems around the world. It's the default "time server" for most of the major Linux distributions and many networked appliances.

Server Status

These servers should be reliable sources of time for your systems; you may view the system's current status by choosing a graph below, or going to the cluster status page.

These servers are running as stratum 2 time servers, with a maximum offset of 10ms.

You may also view the current status of Kirby, Twyla & Samantha's client files, or the pool's running status.

Acceptable use policy

Who may use these servers?

These servers are only intended for use by system administrators configuring NTP clients or Desktop Users configuring their own equipment. They must not be used by equipment manufacturers or systems integrators selling kits preconfigured to use these servers.

  • Desktops and laptops
  • Your own home router
  • Equipment normally found in a server room, such as servers, enterprise routers (not consumer products), SANs etc.

How you may use these servers

  • iburst may be used
  • burst may not be used
  • minpoll/maxpoll may not be changed below the defaults of 64 seconds and 1024 seconds respectively
  • Clients should honor the KOD message
  • Time Server IP addresses may not be hard-coded in any hardware, firmware, or software without advance explicit written permission from the time server operator
  • Time Server DNS addresses should not be hard-coded in any hardware, firmware, or softrware without advance explicit written permission from the time server operator
